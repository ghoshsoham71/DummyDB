# Neo4j (Graph Database)

BurstDB connects to Neo4j via the Bolt protocol and extracts the graph schema — node labels, relationship types, properties, and constraints.

## Ports

Neo4j uses two ports on the user's system:

| Port | Protocol | Purpose |
|------|----------|---------|
| **7474** | HTTP | Neo4j Browser (web UI) |
| **7687** | Bolt | Driver connections (used by BurstDB) |

You can access the Neo4j Browser directly at `http://localhost:7474`.

## Connection

In the **Get Started** page, select **Neo4j (Graph DB)** and provide:

| Field | Default | Description |
|-------|---------|-------------|
| Bolt URI | `bolt://localhost:7687` | Neo4j driver endpoint |
| Username | `neo4j` | Database username |
| Password | *(empty)* | Database password |
| Database | *(default)* | Specific database name (optional) |

## What Gets Extracted

### Nodes (mapped as tables with prefix `Node:`)

- **Labels** — each unique label becomes a "table" entry
- **Properties** — field names and types (via APOC or fallback)
- **Constraints** — UNIQUE, NOT_NULL from `SHOW CONSTRAINTS`
- **Node count** — total nodes per label
- **Outgoing relationships** — relationship type and target label

### Relationships (mapped as tables with prefix `Rel:`)

- **Type** — relationship type name
- **Properties** — any properties on the relationship
- **Source → Target** — label pairs
- **Count** — total relationships of this type

## Example

Given this Cypher data:

```cypher
CREATE (a:Person {name: "Alice", age: 30})
CREATE (b:Person {name: "Bob", age: 25})
CREATE (c:Company {name: "Acme", founded: 2010})
CREATE (a)-[:WORKS_AT {since: 2020}]->(c)
CREATE (a)-[:KNOWS]->(b)
```

BurstDB extracts:

| Table | Type | Properties |
|-------|------|------------|
| `Node:Person` | node | name (VARCHAR), age (INTEGER) |
| `Node:Company` | node | name (VARCHAR), founded (INTEGER) |
| `Rel:WORKS_AT` | relationship | since (INTEGER) |
| `Rel:KNOWS` | relationship | *(none)* |

## Type Mapping

| Neo4j Type | BurstDB Type |
|------------|-------------|
| STRING | VARCHAR |
| INTEGER / LONG | INTEGER / BIGINT |
| FLOAT / DOUBLE | DOUBLE |
| BOOLEAN | BOOLEAN |
| DATE | DATE |
| DATE_TIME | TIMESTAMP |
| DURATION | INTERVAL |
| POINT | GEOMETRY |
| LIST | ARRAY |
| MAP | JSON |

## APOC Plugin

If [APOC](https://neo4j.com/labs/apoc/) is installed, BurstDB uses `apoc.meta.cypher.type()` for precise type inference. Without APOC, types default to VARCHAR using a simpler fallback.

## Docker Quick Start

```bash
docker run -d \
  --name neo4j \
  -p 7474:7474 \
  -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/password \
  neo4j:latest
```

Then connect in BurstDB with:
- URI: `bolt://localhost:7687`
- Username: `neo4j`
- Password: `password`
