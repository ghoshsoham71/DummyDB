# MongoDB (NoSQL)

BurstDB can connect to MongoDB instances, sample documents, and infer the schema automatically.

## How It Works

1. Connect via a MongoDB connection string
2. BurstDB samples documents from each collection (configurable, default: 100)
3. Field types are inferred from the sampled values
4. Indexes and validation rules are extracted
5. The schema is normalized to match the same format as SQL schemas

## Connection

In the **Get Started** page, select **MongoDB (NoSQL)** and provide:

| Field | Example | Required |
|-------|---------|----------|
| Connection String | `mongodb://localhost:27017` | ✅ |
| Database Name | `myApp` | Optional (scans all if blank) |

### Connection String Formats

```
# Local
mongodb://localhost:27017

# With authentication
mongodb://user:password@localhost:27017/myDatabase

# MongoDB Atlas (cloud)
mongodb+srv://user:password@cluster.mongodb.net/myDatabase
```

## Extracted Information

| Feature | Description |
|---------|-------------|
| **Collections** | All non-system collections |
| **Fields** | Inferred from document sampling |
| **Types** | Mapped to standard types (VARCHAR, INTEGER, BOOLEAN, etc.) |
| **Nested fields** | Dot-notation up to 3 levels deep |
| **Indexes** | Unique, compound, and regular indexes |
| **Constraints** | PRIMARY_KEY (_id), UNIQUE, INDEXED, NOT_NULL, NULLABLE |
| **Document count** | Estimated count per collection |

## Type Mapping

| BSON Type | BurstDB Type |
|-----------|-------------|
| String | VARCHAR |
| Int32 / Int64 | INTEGER |
| Double | DOUBLE |
| Boolean | BOOLEAN |
| ObjectId | OBJECT_ID |
| Date | TIMESTAMP |
| Array | ARRAY |
| Object | JSON |
| Binary | BINARY |
| Decimal128 | DECIMAL |

## Limitations

- Schema inference is based on sampling — rare fields may be missed
- Maximum nesting depth is 3 levels
- System collections (`system.*`) are skipped
- `admin`, `local`, and `config` databases are excluded
